

## Раздел 1. Установка ОС, настройка и конфигурирование

### 1. Настройка ядра и служб (Утилита Sysctl)
sudo apt install libreoffice GIMP -y
Для базовой настройки параметров ядра используется утилита `sysctl`. Отредактируем конфигурационный файл:

```bash
sudo nano /etc/sysctl.conf

```

*В конец файла добавьте строку для ускорения работы с памятью:*

```text
vm.swappiness=10

```

Примените настройки ядра:

```bash
sudo sysctl -p

```

### 2. Настройка сетевого протокола SSH

Устанавливаем и активируем службу SSH для удаленного доступа:

```bash
sudo apt update
sudo apt install openssh-server -y
sudo systemctl enable --now ssh

```

### 3. Настройка удаленного доступа к активной сессии

Для Linux Ubuntu проще всего использовать протокол VNC или встроенный RDP:

```bash
sudo apt install xrdp -y
sudo systemctl enable --now xrdp

```

### 4. Настройка интернет-соединения и проверка

Посмотреть имя сетевого интерфейса (например, `enp0s3` или `eth0`):

```bash
ip a

```

Продемонстрировать сетевое соединение преподавателю:

```bash
ping -c 3 google.com

```

### 5. Установка виртуального принтера

Устанавливаем виртуальный PDF-принтер:

```bash
sudo apt install cups printer-driver-cups-pdf -y

```

## Раздел 2. Применение средств защиты компьютерных систем

### 1. Резервное копирование и создание установочного образа

Для создания резервной копии всей системы («слепка») используем встроенную утилиту архивации `tar`:

```bash
sudo tar -czf /backup_fast.tar.gz /etc
```

### 2. Создание точек восстановления (Утилита Timeshift)

Timeshift — стандартный инструмент создания точек восстановления в Ubuntu:

```bash
sudo apt install timeshift -y
sudo timeshift --create --comments "Exam_Point"

```

### 3. Создание групп пользователей и прав доступа

Создаем группу для разработчиков, пользователя и добавляем его в группу:

```bash
sudo groupadd developers
sudo useradd -m -g developers -s /bin/bash dev_user
sudo passwd dev_user

```

Настройка прав доступа к рабочей папке проекта:

```bash
sudo mkdir /projects
sudo chown -R :developers /projects
sudo chmod -R 770 /projects

```

### 4. Настройка аутентификации и журналов мониторинга

Просмотр системного журнала в реальном времени (для демонстрации мониторинга):

```bash
sudo tail -f /var/log/auth.log

```

## Раздел 3. Установка и конфигурирование ПО

### Обоснование выбора программных ресурсов (для устного ответа/документа):

1. **Android Studio** — официальная IDE для разработки под Android устройств, содержит эмулятор Android (AVD) с поддержкой 3D-графики.
2. **Code::Blocks** — кроссплатформенная среда разработки, поддерживает набор компиляторов GCC.
3. **ClamAV + GUICat / Security** — антивирусное ПО для Linux для защиты от угроз.

### Команды установки:

```bash
# Установка Code::Blocks и компиляторов GCC
sudo apt install codeblocks build-essential -y

# Установка Антивируса ClamAV и графической оболочки к нему
sudo apt install clamtk -y

# Установка Android Studio (через snap-пакет)
sudo snap install android-studio --classic

```

### Настройка совместимости и интерфейса

В Linux Ubuntu параметры совместимости графики (низкое разрешение, отключение композиции) регулируются через настройки графической оболочки (GNOME/X11).
Для запуска приложения с принудительным отключением масштабирования на High-DPI экранах используется переменная среды перед запуском (например, для Qt/Gtk программ):

```bash
QT_SCALE_FACTOR=1 codeblocks

```
